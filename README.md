# official-app-hub

本仓库用于归档与发布多个独立 HTML 页面。这些页面以静态文件形式存放，可直接通过浏览器打开或部署到任意静态托管服务中。

---

## 项目简介

`official-app-hub` 是一个通用型的 HTML 页面仓库，主要用途包括：

- 存档多个独立的前端页面（单页应用、工具界面、展示页等）
- 作为集中化的页面发布源，便于版本管理与历史回溯
- 提供统一的结构，方便后续扩展或迁移

本项目不针对特定域名或网站，所有页面内容均为独立单元，彼此之间无强制关联。

---

## 目录说明

仓库目录结构如下（示意）：

```
official-app-hub/
├── pages/           # 存放所有 HTML 页面文件
│   ├── page1/
│   │   └── index.html
│   ├── page2/
│   │   └── index.html
│   └── ...
├── assets/          # 共享资源（如样式、脚本、图片等）
├── README.md        # 本文件
└── .gitignore       # Git 忽略规则
```

- `pages/`：每个子目录代表一个独立的 HTML 页面，内部包含 `index.html` 及该页面所需的配套资源。
- `assets/`：存放可被多个页面引用的通用资源文件，避免重复。

---

## 页面归档说明

- 每个页面应放置于 `pages/` 下的独立子目录中，目录名建议使用有意义的英文或拼音（例如 `tool-calculator`、`dashboard-v1`）。
- 页面内部资源（如 CSS、JS、图片）建议放在该页面的子目录内，或统一引用 `assets/` 中的文件。
- 归档时请确保页面可独立运行，避免依赖外部在线资源（除非必要）。
- 如需更新页面，建议保留旧版本或通过 Git 提交记录追踪变更。
